# Qwen Code 改进建议 — 会话标签与语义搜索 (Session Tags & Agentic Search)

> 核心洞察：随着大模型辅助开发越来越深入日常工作流，用户往往会积攒几十上百个历史会话（Sessions）。目前大部分的 CLI 工具在执行类似 `/resume` 的恢复命令时，仅仅只提供一个冷冰冰的“按最后修改时间排序”的列表。当你一个月后想要找回那条“帮我配置 Redis 哨兵集群”的绝佳 Prompt 记录时，如同大海捞针。Claude Code 在会话管理上引入了类似于 Git 分支理念的 `/tag` 系统，甚至支持用大模型去“语义搜索历史”，让历史会话变成了极其好用的个人知识库；而 Qwen Code 目前在历史找回上缺乏索引。
>
> 返回 [改进建议总览](./qwen-code-improvement-report.md)

## 一、迷失在历史海洋中的痛点

### 1. Qwen Code 现状：基于时间的流水账
如果目前在 Qwen Code 中列出过往会话：
- **痛点一（信息密度极低）**：你看到的可能只是一排 `Session_1702434...` 或者由 AI 自动生成的前几个字的摘要（比如 `关于如何安装 npm`）。如果你的很多任务起手都是“帮我看一下这段代码”，列表里就会充满了一堆不知所云的长相类似的会话名。
- **痛点二（没有聚类机制）**：你上周在 `frontend` 仓库开了 3 个修 UI 的会话，在 `backend` 仓库开了 5 个重构数据库的会话。当你想集中复盘时，系统不支持按仓库（Repo）或业务线进行聚类过滤。

### 2. Claude Code 解决方案：为会话打上高亮标记
在 Claude Code 的 `commands/tag/` 和 `commands/resume/` 的深层源码中，他们把会话管理做到了接近书签管理软件的程度。

#### 机制一：`/tag` 显式标记
用户可以在对话的任意时刻（哪怕是刚开始，或者结束前），在终端里输入：
`/tag bugfix-auth`
这个会话的底层 JSON 描述符里立刻被注入了这个永久的标记。
下一次用户敲出 `/resume` 时，列表的展示极其精美，标签就像 GitHub 的 Label 一样高亮显现在标题右侧，让人一眼就能抓住重点。

#### 机制二：Agentic Session Search (语义化探寻)
这是最科幻的功能！当你在 `/resume` 后面跟上搜索词（如 `/resume docker redis`），系统并不是仅仅去做简单的包含匹配（Substring Match）。
它调用了一个内部的 `agenticSessionSearch` 函数！
它会用一个便宜极速的小模型（比如 Haiku），在后台把过去一个月所有的历史会话的核心对话流快速扫一遍，寻找哪怕标题里没写但实际上讨论了“docker 和 redis”的会话，并按照相关度倒序呈现给用户。

## 二、Qwen Code 的改进路径 (P2 优先级)

帮助用户管理心智负担，是提升日活（DAU）的神兵利器。

### 阶段 1：开发会话元数据（Metadata）打标功能
1. 修改底层 `sessions/` 下保存文件的结构。为 JSONL 头或者对应的配置文件增加一个 `tags: string[]` 的数组和 `customTitle: string`。
2. 开发 `/tag <tag-name>` 命令，允许为当前会话挂载标签。
3. 开发 `/rename <new-name>` 命令，允许用户覆盖掉原本由 AI 生成的废话标题。

### 阶段 2：重写 `/resume` 交互面板
如果 Qwen Code 使用类似 `FuzzyPicker` 或 `SelectInput` 组件：
1. 界面应当支持多列（Columns）布局。
2. 依次展示：`[更新时间]` | `[所属 Git 仓库名]` | `[高亮 Tags]` | `[标题/摘要]`。
3. 允许用户在这个终端组件中输入字母，实时对所有展示出来的行进行 Fuzzy（模糊）正则过滤。

### 阶段 3：引入向量搜索（远期规划）
如果考虑到未来的高阶应用，可以为这些历史会话在本地生成非常轻量级的文本嵌入（Embeddings）存入 SQLite。实现类似于“在本地终端里向你的第二大脑提问”。

## 三、改进收益评估
- **实现成本**：小。核心是几个字符串数组的存取，配合现成的 TUI 选择器，工作量约在百行以内。
- **直接收益**：
  1. **建立极高的切换壁垒**：一旦用户给自己的几百个高价值长对话打满了整齐的标签，Qwen Code 就会变成他的专属云端记事本，他几乎再也无法脱离这个生态。
  2. **跨项目知识沉淀**：解决了由于频繁开新会话带来的“一次性代码”浪费问题，鼓励用户反复回到成功过的最佳实践（Golden Sessions）中去 Copy 代码。
